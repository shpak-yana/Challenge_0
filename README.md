# Challenge_0
Челлендж #0: "Анализ Памяти и Коллекции" (Проверка фундамента)
Напишите простой класс Person (поля: id (int), name (String), age (int)).

В методе main:

Создайте HashMap<Integer, Person>, добавьте 5 объектов Person.
Создайте ArrayList<Person>, добавьте те же 5 объектов.
Создайте второй ArrayList<Person> и добавьте в него новые объекты Person с теми же данными.

Анализ (Ответьте письменно):
1. Как хранятся объекты в памяти при добавлении в HashMap vs ArrayList (первый)?
2. Что происходит при вызове personMap.get(someId)? Как работает хэш-функция?
3. Чем будет отличаться поведение при сравнении list1.get(0) == list2.get(0) и list1.get(0).equals(list2.get(0)) (если equals не переопределен)? Почему?
4. Сколько всего объектов Person создано в куче (Heap) в вашем коде? Объясните.

Ответы:
1. При добавлении объектов в HashMap хранение в памяти реализуется в виде словаря <ключ, значение>. Ключи должны быть уникальны,т.к. указывая ключ, мы можем взаимодействовать со значением(менять, удалять).
ArrayList представляет простой список, аналогичный массиву, за тем исключением, что количество элементов в нем не фиксировано. К элементы можно обращаться как по индексу, так и по значению.
2. Возвращает значение объекта, ключ которого равен k. Если такого элемента не окажется, то возвращается значение null.
3. Ничем. Если не переопределить equals, сравниваться будут объекты в памяти,  а не их поля, которые в данном случае одинаковые.
4. 10 объектов. Первые 5, которые использовались в HashMap и  ArrayList (первый), и вторые 5 - для второго ArrayList.
